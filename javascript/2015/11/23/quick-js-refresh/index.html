A quick javascript refresh

This is a quick javascript knowledge refresh.

  1. In Javascript, we deal with objects. Its an Object Oriented Programming language. We have been using ES 3 standard. Most of us are already using ES5 without knowing. All developer friendly browsers are already supporting ES5.
  2. The primitive data types: number, string, boolean, null, undefined. Functions are also objects here.
  3. There are 2 types of objects – 1. Host Objects 2. Native Objects
  4. Prototypes are used for the inheritance purpose. Each function we create will have prototype property attached to it.
  5. Avoid globals to avoid variable pollutions, NO implied globals & use single “var” pattern
  6. Variable hoisting and functional hoisting
  7. “eval” is evil as the experts say. be careful while using eval, setTimeout, setInterval and Function constructor
  8. parseInt() problem when used without radix. This is taken care in ES5.
  9. Use always object literal to create an object, but you should know about the Object constructor as well.

    10. Example:

    var myObj = {} // object literal
    This is not an empty object, this will go up to the prototype chain while using for .. in.
    To create an empty object,
    var myObj = Object.create(null); // this is purely an empty obejct
    So you dont need to use hasOwnProperty() method to filter the prototype objects.

  10. In case if you dont deliberately return anything from a constructor, it will always return an object NOT “undefined” as the functions do. Constructors “this” object is an instance of that constructor NOT the global “this”. This should be called with new.
  11. Use array literal instead of the Array constructor. There may be an issue with Array constructor.
  12. Functions are the first class objects, If you dont return anything from the function, the “this” will be returned which is a global env. “this”. In ES5, this “this” is no longer pointing to the global “this”. I think, its empty.
  13. If the function declaration defined at the bottom of the scope, the function declaration and definition will be hoisted. That’s why you are free add function declaration anywhere in the code. But if you use, function statement, only the declaration will be hoisted. You should be aware of this.
  14. One should know about Closure
  15. What are Memoization & Currying?
  16. JSlint, YUIDoc
